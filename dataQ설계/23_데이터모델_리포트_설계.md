# 데이터 모델 리포트 설계서

**작성일**: 2026-04-02
**상태**: 설계

---

## 1. 목적

데이터 모델 단위로 표준화 현황, 구조 진단 결과, 품질 지표를 종합한 **리포트를 생성하고 다운로드(PDF/Excel)**할 수 있는 기능.

관리자가 특정 모델의 품질 상태를 한눈에 파악하고, 외부 보고서로 활용할 수 있게 한다.

---

## 2. 메뉴 위치

```
리포트              ← 신규 대메뉴 (아이콘: mdi-file-chart)
  └─ 데이터 모델 리포트
```

대메뉴로 분리하는 이유:
- 기존 메뉴(표준사전/모델/진단 등)는 **관리/실행** 용도
- 리포트는 **조회/출력** 용도로 성격이 다름
- 향후 다른 리포트(표준 사전 리포트, 진단 이력 리포트 등)를 추가할 수 있음

---

## 4. API 설계

| Method | URL | 설명 |
|--------|-----|------|
| POST | `/api/report/dataModelReport` | 리포트 데이터 조회 (화면용) |
| POST | `/api/report/dataModelReportExcel` | Excel 다운로드 |
| POST | `/api/report/dataModelReportPdf` | PDF 다운로드 |

### 요청 파라미터
```json
{
  "dataModelId": "모델 ID",
  "clctId": "수집 ID (선택, 없으면 최신)"
}
```

### 응답 (화면용)
```json
{
  "modelInfo": { ... },
  "diagSummary": { "stndRate": 19.9, "totalCnt": 422, "matchCnt": 84, ... },
  "tableStats": [ { "tableName": "TB_USER", "columnCnt": 15, "matchCnt": 8, "rate": 53.3 }, ... ],
  "nonStandardColumns": [ { "tableName": "TB_USER", "columnName": "HP_NO", "diagType": "TERM_NOT_EXIST" }, ... ],
  "structDiagResult": { ... },
  "diagTrend": [ { "diagDt": "03-31", "stndRate": 19.9, "resultCnt": 338 }, ... ]
}
```

## 5. 데이터 소스

리포트에 필요한 데이터는 **기존 API/테이블에서 모두 조회 가능**:

| 섹션 | 데이터 소스 |
|------|-----------|
| 모델 개요 | TB_DATA_MODEL + TB_DATA_MODEL_STATS |
| 표준화 진단 요약 | TB_DIAG_JOB (최신 DONE) |
| 테이블별 준수율 | TB_DIAG_RESULT GROUP BY OBJ_NM |
| 비표준 컬럼 | TB_DIAG_RESULT |
| 구조 진단 | TB_STRUCT_DIAG_HISTORY |
| 진단 추이 | TB_DIAG_JOB 최근 5건 |

---

## 6. Excel 출력

Apache POI (기존 ExcelDownloadService 활용):
- 시트 1: 모델 개요 + 진단 요약
- 시트 2: 테이블별 준수율
- 시트 3: 비표준 컬럼 목록
- 시트 4: 구조 진단 결과

---

## 7. PDF 출력

Selenium Edge WebDriver (이미 설치됨):
- 리포트 HTML을 생성 → Edge headless로 PDF 변환
- 또는 iText/Flying Saucer 라이브러리 사용

현실적 방안: **HTML → Edge CDP `Page.printToPDF`** (이미 매뉴얼 PDF에서 사용한 방식)
