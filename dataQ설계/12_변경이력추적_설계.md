# 변경 이력 추적 설계서

**작성일**: 2026-04-01
**상태**: 설계 (미구현)

---

## 1. 개요

### 1.1 목적
단어/용어/도메인/코드의 등록/수정/삭제 이력을 기록하여, **누가 언제 무엇을 어떻게 변경했는지** 추적 가능하게 한다.

### 1.2 현재 상태
- 승인/결재 로그만 존재 (승인 여부만 기록)
- 실제 값의 변경 내용(이전값 → 변경값)은 추적 불가
- 일괄 업로드/등록 시 어떤 데이터가 들어갔는지 사후 확인 불가

### 1.3 경쟁사 대비
- SMETA: 모든 항목에 이력 저장 + 이력 간 비교 기능
- WISE META: 변경 이력 자동 추적
- DataQ: 미지원 → **Phase 1 과제**

## 2. 설계 원칙

### 2.1 이력 저장 단위
- **개별 변경** (1건 수정/삭제): 이력 마스터 1행
- **일괄 등록/업로드** (N건): 이력 마스터 **1행** + 상세 테이블에 N행

### 2.2 이력 대상

| 대상 | 추적 항목 |
|------|----------|
| **단어** (TB_WORD) | 등록, 수정(한글명/영문명/영문약어/설명/도메인분류), 삭제 |
| **용어** (TB_TERMS) | 등록, 수정(한글명/영문약어/도메인/설명), 삭제 |
| **도메인** (TB_DOMAIN) | 등록, 수정(도메인명/데이터타입/길이/설명), 삭제 |
| **코드** (TB_CODE_INFO) | 등록, 수정, 삭제 |
| **코드 데이터** (TB_CODE_DATA) | 등록, 수정, 삭제 |

### 2.3 이력 저장 시점

| 동작 | 이력 유형 | 마스터 행 수 | 상세 행 수 |
|------|----------|-----------|----------|
| 화면에서 1건 등록 | INSERT | 1 | 0 |
| 화면에서 1건 수정 | UPDATE | 1 (이전값/변경값 포함) | 0 |
| 화면에서 1건 삭제 | DELETE | 1 | 0 |
| 엑셀 일괄 업로드 (N건) | BULK_INSERT | 1 (요약: "단어 일괄등록 N건") | N |
| 표준화 추천 일괄등록 (N건) | BULK_INSERT | 1 | N |
| 진단결과 일괄 용어등록 (N건) | BULK_INSERT | 1 | N |

## 3. 테이블 설계

### 3.1 TB_CHANGE_HISTORY (이력 마스터)

```sql
CREATE TABLE TB_CHANGE_HISTORY (
    CHANGE_ID         VARCHAR(40)     NOT NULL,
    CHANGE_TYPE       VARCHAR(20)     NOT NULL,   -- INSERT, UPDATE, DELETE, BULK_INSERT
    TARGET_TYPE       VARCHAR(20)     NOT NULL,   -- WORD, TERM, DOMAIN, CODE, CODE_DATA
    TARGET_ID         VARCHAR(40),                -- 대상 레코드 ID (개별 변경 시)
    TARGET_NM         VARCHAR(200),               -- 대상명 (예: 단어명, 용어명)
    CHANGE_CNT        INTEGER         DEFAULT 1,  -- 변경 건수 (일괄 시 N)
    SUMMARY           VARCHAR(500),               -- 요약 (예: "단어 일괄등록 32건")
    PREV_VALUE        TEXT,                       -- 이전 값 (JSON, 개별 변경 시)
    CURR_VALUE        TEXT,                       -- 변경 값 (JSON, 개별 변경 시)
    CHANGE_USER_ID    VARCHAR(40),                -- 변경자
    CHANGE_DT         TIMESTAMP       DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT PK_TB_CHANGE_HISTORY PRIMARY KEY (CHANGE_ID)
);

CREATE INDEX IDX_CHANGE_HISTORY_DT ON TB_CHANGE_HISTORY (CHANGE_DT DESC);
CREATE INDEX IDX_CHANGE_HISTORY_TARGET ON TB_CHANGE_HISTORY (TARGET_TYPE, CHANGE_DT DESC);
```

### 3.2 TB_CHANGE_HISTORY_DETAIL (일괄 상세)

일괄 등록/업로드 시에만 사용. 개별 변경에는 사용하지 않음.

```sql
CREATE TABLE TB_CHANGE_HISTORY_DETAIL (
    CHANGE_ID         VARCHAR(40)     NOT NULL,
    SEQ               INTEGER         NOT NULL,
    TARGET_ID         VARCHAR(40),                -- 개별 레코드 ID
    TARGET_NM         VARCHAR(200),               -- 대상명
    DETAIL_TYPE       VARCHAR(20),                -- INSERT, UPDATE, DELETE, SKIP, FAIL
    PREV_VALUE        TEXT,                       -- 이전 값 (JSON, 수정 시)
    CURR_VALUE        TEXT,                       -- 변경 값 (JSON)
    REMARK            VARCHAR(500),               -- 비고 (실패사유, 스킵사유 등)
    CONSTRAINT PK_TB_CHANGE_HISTORY_DETAIL PRIMARY KEY (CHANGE_ID, SEQ)
);
```

### 3.3 PREV_VALUE / CURR_VALUE 형식 (JSON)

```json
// 단어 수정 예시
{
  "wordNm": "사용자",
  "wordEngNm": "USER",
  "wordEngAbrvNm": "USR",
  "wordDesc": "시스템을 사용하는 사람",
  "domainClsfNm": ""
}
```

개별 변경 시: 마스터의 `PREV_VALUE`에 수정 전 값, `CURR_VALUE`에 수정 후 값
등록 시: `PREV_VALUE` = null, `CURR_VALUE` = 등록된 값
삭제 시: `PREV_VALUE` = 삭제 전 값, `CURR_VALUE` = null

단어/용어/도메인 상세 조회 시 "이력 보기" 버튼 → 해당 항목의 변경 이력만 필터링하여 표시.

```

  영문약어: USR
  ...
  [이력 보기] ← 클릭 시 이 단어의 변경 이력만 표시
```

## 5. 이력 저장 로직

### 5.1 개별 변경 (INSERT/UPDATE/DELETE)

```java
// Controller 또는 Service에서 변경 전 기존 값 조회
Map<String, Object> prevValue = selectCurrentValue(targetType, targetId);

// 실제 변경 수행
performUpdate(targetType, targetId, newValue);

// 이력 저장
Map<String, Object> history = new HashMap<>();
history.put("changeId", StringUtils.getUUID());
history.put("changeType", "UPDATE");
history.put("targetType", "WORD");
history.put("targetId", targetId);
history.put("targetNm", wordNm);
history.put("changeCnt", 1);
history.put("summary", "단어 수정: " + wordNm);
history.put("prevValue", toJson(prevValue));
history.put("currValue", toJson(newValue));
history.put("changeUserId", userId);
sqlSessionTemplate.insert("changehistory.insertChangeHistory", history);
```

### 5.2 일괄 등록 (BULK_INSERT)

```java
// 일괄 등록 수행
int successCnt = 0, skipCnt = 0, failCnt = 0;
String changeId = StringUtils.getUUID();
List<Map<String, Object>> details = new ArrayList<>();

for (Item item : items) {
    Map<String, Object> detail = new HashMap<>();
    detail.put("changeId", changeId);
    detail.put("seq", seq++);
    detail.put("targetNm", item.getName());
    
    try {
        if (isDuplicate(item)) {
            detail.put("detailType", "SKIP");
            detail.put("remark", "이미 등록됨");
            skipCnt++;
        } else {
            performInsert(item);
            detail.put("detailType", "INSERT");
            detail.put("currValue", toJson(item));
            successCnt++;
        }
    } catch (Exception e) {
        detail.put("detailType", "FAIL");
        detail.put("remark", e.getMessage());
        failCnt++;
    }
    details.add(detail);
}

// 마스터 1건
Map<String, Object> history = new HashMap<>();
history.put("changeId", changeId);
history.put("changeType", "BULK_INSERT");
history.put("targetType", "TERM");
history.put("changeCnt", successCnt + skipCnt + failCnt);
history.put("summary", String.format("용어 일괄등록 %d건 (성공:%d, 스킵:%d, 실패:%d)", 
    successCnt + skipCnt + failCnt, successCnt, skipCnt, failCnt));
history.put("changeUserId", userId);
sqlSessionTemplate.insert("changehistory.insertChangeHistory", history);

// 상세 N건
for (Map<String, Object> detail : details) {
    sqlSessionTemplate.insert("changehistory.insertChangeHistoryDetail", detail);
}
```

## 7. 구현 우선순위

| 순서 | 항목 | 필수 여부 |
|------|------|----------|
| 1 | DDL (TB_CHANGE_HISTORY, DETAIL) + MyBatis 매퍼 | 필수 |
| 2 | 개별 변경 이력 저장 (UPDATE/DELETE에 이전값 포함) | 필수 |
| 3 | 일괄 등록 이력 저장 (마스터 1건 + 상세 N건) | 필수 |
| 4 | 이력 조회 화면 (필터 + 상세) | 필수 |
| 5 | 각 관리 화면에서 "이력 보기" 버튼 | 권장 |
| 6 | 이력 간 비교 기능 (SMETA 수준) | 선택 |
