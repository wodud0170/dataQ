**작성일**: 2026-04-02

## 1. 현재 문제

### Greedy 알고리즘의 한계

현재 `greedySplit`은 **왼쪽부터 가장 긴 매칭을 욕심껏 선택**합니다.

```
"테슬라코일제목"
  → OKT: [테슬라코일제, 목, 테슬라, 코일, 제목, ...]
  → greedy: "테슬라코일제"(OKT 6글자) 먼저 매칭 → 나머지 "목"(1글자)
  → 결과: [테슬라코일제(UNRECOGNIZED), 목(UNRECOGNIZED)]
  → "제목"(TB_WORD)이 있는데 못 찾음
```

### 근본 원인
- Greedy는 왼쪽에서 결정을 내리면 되돌릴 수 없음
- 뒤에 등록 단어가 있어도, 앞에서 OKT 토큰이 먼저 먹어버리면 끝

## 2. DP 알고리즘 설계

### 2.1 핵심 아이디어

입력 문자열의 **모든 위치에서 모든 가능한 토큰을 시도**하고, **총 점수가 가장 높은 분리를 선택**.

### 2.2 점수 체계

| 토큰 유형 | 점수 | 이유 |
|----------|------|------|
| TB_WORD 등록 단어 | **10000** + 길이×100 | 최우선. 긴 등록 단어가 짧은 것보다 좋음 |
| TB_WORD_DICT 추천 사전 | **5000** + 길이×100 | 등록 다음 우선 |
| OKT 토큰 (2~4글자) | **500** + 길이×50 | 적당한 길이의 일반 명사 |
| OKT 토큰 (5글자+) | **100** | 복합어일 가능성 높음, 낮은 점수 |
| OKT 토큰 (1글자) | **10** | 의미 단위로 약함 |
| 매칭 안 되는 1글자 스킵 | **-500** | 페널티 (최대한 피하도록) |

### 2.3 DP 테이블

```
입력: "테슬라코일제목" (n = 7)

dp[i] = 위치 0~i를 분리했을 때 얻을 수 있는 최대 점수
parent[i] = 위치 i에서 어떤 토큰이 끝나는지 (역추적용)

dp[0] = 0 (시작)
dp[7] = ? (끝 = 최적 분리의 총 점수)

각 위치 i에서:
  모든 후보 토큰 t가 input[i..i+len(t)]에 매칭되면:
    dp[i + len(t)] = max(dp[i + len(t)], dp[i] + score(t))
    parent[i + len(t)] = (i, t)
```

### 2.4 예시: "테슬라코일제목"

```
후보 토큰 (OKT + TB_WORD):
  OKT: 테슬라코일제(6), 테슬라(3), 코일(2), 제목(2), 테(1), 슬(1), ...
  TB_WORD: 제목(2, 점수 10200)
  TB_WORD_DICT: (해당 없음)

dp 계산:
  dp[0] = 0
  
  위치 0에서:
    "테슬라코일제"(OKT 6글자) → dp[6] = 0 + 100 = 100
    "테슬라"(OKT 3글자) → dp[3] = 0 + 650 = 650
    "테"(OKT 1글자) → dp[1] = 0 + 10 = 10

  위치 3에서 (dp[3]=650, "테슬라" 뒤):
    "코일"(OKT 2글자) → dp[5] = 650 + 600 = 1250
    "코일제"(OKT 3글자) → dp[6] = 650 + 650 = 1300
    "코일제목"(OKT 4글자) → dp[7] = 650 + 700 = 1350

  위치 5에서 (dp[5]=1250, "테슬라코일" 뒤):
    "제목"(TB_WORD!) → dp[7] = 1250 + 10200 = 11450 ✨ 최고점!
    "제"(OKT 1글자) → dp[6] = 1250 + 10 = 1260

  위치 6에서 (dp[6]=1300):
    "목"(OKT 1글자) → dp[7] = 1300 + 10 = 1310

최종: dp[7] = 11450 (역추적: 테슬라 + 코일 + 제목)
```

**결과: [테슬라, 코일, 제목]** — "제목"이 TB_WORD로 정상 매칭!

## 3. 구현 상세

### 3.1 메서드 시그니처

```java
/**
 * DP 기반 최적 단어 분리
 * 
 * @param input 입력 문자열 (공백 제거됨)
 * @param candidates OKT 토큰 후보 세트
 * @param wordsByNm TB_WORD 등록 단어 맵
 * @param wordDict TB_WORD_DICT 추천 사전 맵
 * @return 최적 분리된 토큰 목록
 */
private List<String> dpSplit(String input, Set<String> candidates,
        Map<String, List<StdWordVo>> wordsByNm,
        Map<String, Map<String, Object>> wordDict)
```

### 3.2 의사코드

```
function dpSplit(input, candidates, wordsByNm, wordDict):
    n = input.length
    dp = new int[n+1], fill with -∞
    parent = new int[n+1], fill with -1  // 이전 위치
    token = new String[n+1]              // 해당 위치에서 선택된 토큰
    dp[0] = 0
    
    for i = 0 to n-1:
        if dp[i] == -∞: continue  // 도달 불가
        
        // 1. 모든 후보 토큰 시도
        for each candidate in candidates:
            if input[i..].startsWith(candidate) AND candidate != input:
                score = calculateScore(candidate, wordsByNm, wordDict)
                if dp[i] + score > dp[i + candidate.length]:
                    dp[i + candidate.length] = dp[i] + score
                    parent[i + candidate.length] = i
                    token[i + candidate.length] = candidate
        
        // 2. 1글자 스킵 (매칭 안 될 때 대비)
        if dp[i] - 500 > dp[i+1]:
            dp[i+1] = dp[i] - 500
            parent[i+1] = i
            token[i+1] = input[i..i+1]
    
    // 역추적
    result = []
    pos = n
    while pos > 0:
        result.add(0, token[pos])
        pos = parent[pos]
    
    return result
```

### 3.3 점수 계산

```java
private int calculateScore(String token, Map wordsByNm, Map wordDict) {
    if (wordsByNm.containsKey(token)) {
        return 10000 + token.length() * 100;  // TB_WORD
    }
    if (wordDict.containsKey(token)) {
        return 5000 + token.length() * 100;   // DICT
    }
    // OKT 토큰
    if (token.length() >= 2 && token.length() <= 4) {
        return 500 + token.length() * 50;     // 적당한 길이
    }
    if (token.length() >= 5) {
        return 100;                            // 복합어 가능성
    }
    return 10;                                 // 1글자
}
```

## 4. 기존 코드 변경

### 4.1 DataStandardController.java

`greedySplit` → `dpSplit`으로 교체. `generateAlternativeSplit`은 유지 (2순위는 DP 결과를 sub-split).

```
현재:
  List<String> primaryTokens = greedySplit(input, candidates, wordsByNm, wordDict);

변경:
  List<String> primaryTokens = dpSplit(input, candidates, wordsByNm, wordDict);
```

### 4.2 greedySplit 유지 여부

`greedySplit`은 삭제하지 않고 `dpSplit`이 실패할 경우의 fallback으로 유지.
