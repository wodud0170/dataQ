# dataQ 개발 가이드

## 1. 프로젝트 구조

```
ndata-quality-master/
├── q-center/        웹 서버 (Spring Boot + Vue.js)  — Java 29개, Vue 42개
├── q-executor/      백그라운드 워커 (Spring Boot)     — Java 15개
├── q-common/        공유 라이브러리 (Model, Service)  — Java 29개, MyBatis 매퍼 12개
└── dataQ설계/       설계 문서
```

## 2. 아키텍처 다이어그램

```
Browser (Vue.js SPA)
    |
    | HTTP REST / WebSocket
    v
q-center (Spring Boot Web Server, port 8080)
    |                          |
    | MyBatis                  | WebClient (HTTP)
    v                          v
PostgreSQL (품질 DB)      q-executor (Spring Boot Worker, port 8081)
                               |
                               | JDBC (DBHandler)
                               v
                          Target DBMS (Oracle, PostgreSQL, MariaDB, MSSQL, Cubrid)

WebSocket (STOMP):
  q-center <---> q-executor (수집/진단 진행 상황 실시간 전송)
```

## 4. 주요 데이터 흐름

### 표준 등록 (단어/용어/도메인)
```
화면(Vue) → DataStandardController → MyBatis → PostgreSQL
  → [관리자] 즉시 승인 + 변경이력 저장
  → [일반 사용자] 승인 요청 → 관리자 승인 → APRV_YN='Y' + 변경이력 저장
```

### 표준화 진단
```
화면(Vue) → DiagController.startDiag()
  → TB_DIAG_JOB 생성 (READY)
  → q-executor DiagController → DiagService.run()
    → 전체 용어 메모리 로드
    → 컬럼별 진단 (용어 존재/한글명/타입/길이)
    → TB_DIAG_RESULT에 결과 저장
    → TB_DIAG_JOB 상태 갱신 (DONE)
화면에서 폴링으로 진행 상황 확인
```

### 데이터모델 수집
```
화면(Vue) → DataModelController.collectDataModel()
  → q-executor DataModelService.collectDataModel()
    → 대상 DBMS 접속 (DBHandler)
    → 스키마 필터 적용
    → 테이블 메타(OBJ) 수집 → TB_DATA_MODEL_OBJ 저장
    → 컬럼 메타(ATTR) 수집 → TB_DATA_MODEL_ATTR 저장
    → 통계(STATS) 저장
    → WebSocket으로 진행 상황 실시간 전송
```

### 구조 진단
```
화면(Vue) → StructDiagController.execute()
  → TB_STRUCT_DIAG_HISTORY 생성 (READY)
  → q-executor StructDiagService.run()
    → 수집 스냅샷(prev) 로드
    → 실제 DBMS(curr) 접속하여 현재 스키마 읽기
    → prev vs curr Diff: ADDED / DELETED / MODIFIED
    → TB_STRUCT_DIAG_DETAIL에 변경 건 저장
    → TB_DATA_MODEL.structDiagYn 갱신
```

## 6. 빌드 및 실행 방법

### 사전 요구사항
- JDK 11+
- Maven 3.6+
- Node.js 14+ (Vue 프론트엔드 빌드 시)
- PostgreSQL 12+

### 빌드

```bash
# 1. q-common (공유 라이브러리) 빌드 및 로컬 설치
cd q-common
mvn install -DskipTests

# 2. q-center (웹 서버) 빌드
cd q-center
mvn compile -pl . -Dfrontend.skip=true    # Java만
mvn package -pl . -DskipTests             # JAR/WAR 패키징

# 3. q-executor (워커) 빌드
cd q-executor
mvn compile -pl .
mvn package -pl . -DskipTests

# 4. Vue 프론트엔드 빌드 (선택)
cd q-center/vue/front
npm install
npm run build
```

### 실행

```bash
# q-executor 먼저 실행 (Worker)
cd q-executor
mvn spring-boot:run

# q-center 실행 (Web Server)
cd q-center
mvn spring-boot:run -Dfrontend.skip=true
```

- q-center: http://localhost:8080
- q-executor: http://localhost:8081

### 환경 설정
- `q-center/src/main/resources/application.yml` — DB 접속, STOMP 브로커 등
- `q-executor/src/main/resources/application.yml` — DB 접속, STOMP 클라이언트 등
- `q-common/src/main/resources/application-datasource.yml` — 공통 DB 설정
