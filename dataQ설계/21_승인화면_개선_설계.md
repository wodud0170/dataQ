# 승인 화면 개선 + 내 요청 현황 설계서

**작성일**: 2026-04-02
**상태**: 설계

---

## 1. 현재 문제

### 1.1 일반 사용자 관점
- 단어/용어/도메인을 등록 요청한 후 **결과를 확인할 곳이 없음**
- "내가 신청한 게 승인됐는지, 반려됐는지" 알 수 없음
- 반려 시 사유도 확인 불가

### 1.2 관리자 관점
- 승인 화면이 테이블 하나에 모든 상태가 섞여 있음
- 승인 요청/검토/완료/반려가 한 목록에 혼재
- 상세 정보 확인이 불편 (항목명+영문명만 보임)
- 일괄 승인/반려 UX가 직관적이지 않음

## 2. 메뉴 구조 변경

```
현재:
  관리
    ├─ 사용자 관리
    ├─ 승인          ← 관리자 전용
    ├─ 데이터 소스
    └─ 시스템

변경:
  내 요청              ← 신규 대메뉴 (모든 사용자)
    └─ 내 요청 현황

  관리
    ├─ 사용자 관리
    ├─ 승인 관리       ← UI 개선
    ├─ 데이터 소스
    └─ 시스템
```

## 3. 내 요청 현황 (신규)

### 3.1 대상 사용자
- 모든 사용자 (관리자 포함)
- 로그인한 사용자가 등록 요청한 항목만 표시

### 3.2 화면 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│  내 요청 현황                                              │

│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 항목명     │ 영문명    │ 유형  │ 상태    │ 요청일   │ 처리일 │  │
│  │ 사용자이름  │ USR_NM  │ 용어  │ 🟢 승인 │ 04-01  │ 04-02│  │

### 3.3 상태별 색상

| 상태 | 칩 색상 | 설명 |
|------|--------|------|
| 승인대기 | 🟡 주황 | APRV_YN = 'N', 아직 처리 안 됨 |
| 승인완료 | 🟢 초록 | APRV_YN = 'Y' |
| 반려 | 🔴 빨강 | TB_APRV_STATS.APRV_STAT = 3 (REJECTED) |

### 3.4 상세 정보

클릭 시 하단에 표시:
- **단어**: 한글명, 영문약어, 영문명, 설명, 단어유형, 도메인분류
- **용어**: 한글명, 영문약어, 도메인, 데이터타입/길이, 구성단어 목록
- **도메인**: 도메인명, 그룹, 분류, 데이터타입, 길이

반려 시:
- 반려 사유 표시
- 반려일시, 반려자 표시

### 3.5 API

| Method | URL | 설명 |
|--------|-----|------|
| POST | `/api/std/getMyRequestList` | 내 요청 목록 (로그인 사용자 기준) |

기존 `selectStdAprvStatList` 쿼리에 `reqUserId` 파라미터를 전달하면 됨.

## 4. 승인 관리 UI 개선 (기존 화면)

### 4.1 현재 UI 문제

```
현재: 
  [테이블: 항목명 | 영문명 | 유형 | 상태 | 요청자 | 요청일 | 승인자 | 처리일 | 사유]
  → 상태가 섞여서 뭐가 승인대기인지 한눈에 안 보임
  → 상세 보려면 어딘가를 클릭해야 하는데 불명확

### 4.3 개선 포인트

| 항목 | 현재 | 개선 |
|------|------|------|
| 상태 구분 | 한 목록에 혼재 | **상태별 탭/칩 필터** (기본: 승인대기) |
| 상세 정보 | 목록에서 안 보임 | **클릭 시 하단 패널** + 유형별 상세 |
| 승인/반려 | 상단 버튼 (어떤 항목인지 불명확) | **상세 패널 내 승인/반려** + 반려 사유 입력 |
| 일괄 처리 | 있지만 UX 불편 | **체크박스 + 하단 일괄 버튼** |
| 반려 사유 | 입력 가능하지만 안 보임 | **반려 시 사유 필수 입력** 강조 |

## 5. 데이터 흐름

### 5.1 일반 사용자

```
사용자가 단어/용어/도메인 등록
    │
    ├─ 관리자: APRV_YN = 'Y' → 바로 사용 가능 (내 요청에는 표시 안 됨)
    │
    └─ 일반 사용자: APRV_YN = 'N' → 승인대기
         │
         ├─ "내 요청 현황"에서 상태 확인 가능
         │
         ├─ 관리자가 승인 → APRV_YN = 'Y' + 이력 저장
         │   └─ 사용자가 "내 요청"에서 승인완료 확인
         │
         └─ 관리자가 반려 → TB_APRV_STATS에 반려 사유 저장
             └─ 사용자가 "내 요청"에서 반려 사유 확인
```

### 5.2 관리자

```
"승인 관리" 접속
    │
    ├─ 승인대기 탭 (기본) → 처리할 항목 목록
    │
    ├─ 항목 클릭 → 상세 확인
    │
    ├─ 개별 승인/반려 (상세 패널)
    │
    └─ 일괄 승인/반려 (체크박스 선택 후)
```

## 6. API

### 6.1 기존 API (변경 없음)

| API | 용도 |
|-----|------|
| `POST /api/std/getApprovalList` | 승인 목록 조회 (기존) |
| `POST /api/std/putStdAprvStat` | 승인/반려 처리 (기존) |

### 6.2 신규 API

| Method | URL | 설명 |
|--------|-----|------|
| POST | `/api/std/getMyRequestList` | 내 요청 목록 (로그인 사용자의 미승인 항목) |

기존 `selectStdAprvStatList`에 `reqUserId = sessionService.getUserId()`를 전달하면 됨.
추가로 승인 완료된 항목도 보여주려면 별도 쿼리 필요.
