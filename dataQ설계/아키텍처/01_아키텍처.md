# 01. 전체 아키텍처

> 최종 수정: 2026-03-23

## 시스템 구조도

```
┌─────────────────────────────────────────────────────────────┐
│                        브라우저 (사용자)                      │
│                   Vue.js 2.x + Vuetify                       │
└─────────────────────┬───────────────────────────────────────┘
                      │ HTTP (REST API)
                      │ WebSocket (STOMP)
┌─────────────────────▼───────────────────────────────────────┐
│                    q-center (포탈)                           │
│          Spring Boot 2.7.10 + Spring Security               │
│          PORT: 8080                                          │
│                                                              │
│  Controller → (MyBatis) → PostgreSQL (내부 DB)              │
│       │                                                      │
│       └──── WebClient ────────────────────────────────────┐ │
└──────────────────────────────────────────────────────────┐│ │
                                                           ││ │
┌──────────────────────────────────────────────────────────▼▼─┐
│                  q-executor (백그라운드)                      │
│          Spring Boot 2.7.10                                  │
│          PORT: 8090                                          │
│                                                              │
│  Controller → Service (Runnable) → 외부 DB (Oracle 등)      │
│                    │                                         │
│                    └── (MyBatis) → PostgreSQL (내부 DB)      │
└──────────────────────────────────────────────────────────────┘

내부 DB: PostgreSQL (표준 사전, 수집 결과, 진단 결과 저장)
외부 DB: Oracle, PostgreSQL, MySQL, MariaDB, MSSQL, Cubrid (수집 대상)
```

## 모듈별 역할

| 모듈 | 역할 | 비고 |
|------|------|------|
| **q-common** | VO 클래스, MyBatis Mapper XML, 유틸리티 | JAR로 빌드 후 나머지 모듈이 의존 |
| **q-center** | 웹 포탈 (API 서버 + Vue 프론트엔드) | Spring Security 인증 포함 |
| **q-executor** | 백그라운드 배치 처리 (수집, 진단) | q-center에서 WebClient로 호출 |

## q-center ↔ q-executor 통신

```
q-center (DataModelController)
  │
  ├── POST http://q-executor:8090/api/dm/collectDataModel
  ├── POST http://q-executor:8090/api/std/upload*
  └── POST http://q-executor:8090/api/diag/runDiag

q-executor 처리 방식:
  - startService(Runnable): 별도 스레드에서 비동기 실행
  - runService(Runnable):   동기 실행 (HTTP 응답으로 완료 확인)
```

> **주의**: 업로드 처리는 `runService` (동기) 사용.
> WebSocket 완료 메시지가 장시간 DB 처리 후 유실되는 문제로 인해 HTTP 응답을 완료 신호로 사용.

## WebSocket 구조

```
브라우저 (SockJS + STOMP)
  │
  ├── 연결: ws://host/sockjs
  ├── 구독 (전체): /topic/onMessage
  └── 구독 (개인): /user/exchange/amq.direct/message.window

q-center (WebSocketController)
  └── q-executor → StompSessionService → /topic/onMessage
```

## 보안

- **인증**: Spring Security Form Login (JSESSIONID 세션)
- **인가**: 대부분 `permitAll()` (내부 시스템), 일부 관리 API만 인증 필요
- **CSRF**: Disabled (개발 편의)
- **비밀번호 암호화**: Jasypt
- **DB 접속 정보 암호화**: Jasypt로 암호화하여 저장

## 빌드/배포 순서

```
1. cd q-common && mvn install     # q-common JAR을 로컬 Maven 저장소에 설치
2. cd q-center && mvn package     # Vue 빌드 + Spring Boot JAR 생성
3. cd q-executor && mvn package   # Spring Boot JAR 생성
4. java -jar q-center.jar
5. java -jar q-executor.jar
```
