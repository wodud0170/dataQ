### 탭 생명주기

```
1. 네비게이션 클릭
   → NdNav: $emit('addTabItem', title, name)
   → NdLayout: tabs 배열에 { title, name, key:UUID } 추가
   → tabs watch → matchContent() → activeContent = name

2. 탭 버튼 클릭
   → NdContent: $emit('addActiveContent', name, index)
   → NdLayout: activeContent = name

3. 탭 닫기 (X 버튼)
   → NdContent: $emit('removeTabItem', index, title, name)
   → NdLayout: tabs 배열에서 제거 → 인접 탭 활성화

4. 탭 드래그 (순서 변경)
   → vue-draggable 이벤트
   → NdContent: $emit('changeTabItem', items)
   → NdLayout: tabs 배열 순서 재정렬
```

### 이벤트 흐름

```
NdNav ──$emit──▶ NdLayout ──props──▶ NdContent
                    │
                    └─────props──▶ NdNav (tabs, activeContent, navGroup 상태)
```

## 컴포넌트 상세

### main.js 핵심 설정

```js
// API URL prefix (모든 컴포넌트에서 this.$APIURL.base 사용)
Vue.prototype.$APIURL = { base: 'http://host:port/' }

// WebSocket 연결 (세션 확인 후)
const stompClient = Stomp.over(new SockJS('/sockjs'));
stompClient.connect({}, () => {
  // 전체 공지 구독
  stompClient.subscribe('/topic/onMessage', handler)
  // 개인 메시지 구독
  stompClient.subscribe('/user/exchange/amq.direct/message.window', handler)
});
```

### NdLayout.vue 핵심 데이터

```js
data() {
  return {
    tabs: [],              // 열린 탭 목록 [{title, name, key}]
    activeContent: '',     // 현재 활성 탭 ID
    // 네비게이션 그룹 확장 상태
    navDsGroup: null,
    navDmGroup: null,
    navDiagGroup: null,
    navMmGroup: null,
    navApprovalStatus: 0,  // 대시보드 ↔ 승인 탭 연동
  }

```

## 화면별 특이사항

### DSDataDiag (진단 실행)
- 데이터모델명 autocomplete → `getDataModelStatsList` API 사용
- 수집일시 select → `getDataModelClctList` API (1년 범위)
- 진단 시작 후 3초 폴링 (`setInterval`) → `getDiagJobById`
- RUNNING 상태 진행바 표시
- DONE/STOPPED/ERROR → 폴링 중단

### DSDataDiagResult (진단 결과)
- 진단 이력 select → 선택 시 결과 자동 로드
- 두 탭: "테이블 집계" / "컬럼 상세"
- 컬럼 상세 테이블: 필터 변경 시 `:key` 변경으로 페이징 리셋
- 검색조건 대소문자 구분 없음 (`.toLowerCase()` 처리)

### DSTerm / DSWord / DSDomain 등 (표준 사전)
- 일괄등록(Excel 업로드): `runService` (동기) → HTTP 응답으로 완료 확인
- 완료 후 `.then()` 핸들러에서 로딩 해제 및 목록 새로고침

### DSDatamodelStatus (데이터모델 현황)
- 상단 필터에서 데이터모델 선택 → 하위 컴포넌트(StatusTable, StatusColumn)에 데이터 전달

## API 호출 패턴

```js
// 공통 패턴
const url = this.$APIURL.base + 'api/{module}/{endpoint}';

// GET 파라미터
axios.get(url, { params: { key: value } })

// POST body
axios.post(url, { key: value })

// 파일 업로드
const formData = new FormData();
formData.append('file', file);
axios.post(url, formData, { headers: { 'Content-Type': 'multipart/form-data' } })
  .then(res => {
    if (res.data.resultCode === 200) { /* 성공 */ }
  })
```

## 주의사항

1. **API URL prefix**: `this.$APIURL.base` 없으면 요청이 엉뚱한 경로로 감
2. **응답 데이터**: `res.data.contents` (not `res.data.data`)
3. **Vuetify v-tabs**: `activeTab` 초기값 `0` 필수. `v-tabs-items`와 세트로 사용
4. **페이징 리셋**: 필터 변경 시 `:key` 바인딩으로 강제 리마운트
5. **WebSocket 완료 의존 금지**: 업로드 완료는 HTTP `.then()`으로 처리
