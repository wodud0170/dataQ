# 승인 프로세스 강화 설계서

**작성일**: 2026-04-01
**상태**: 설계 (미구현)

---

## 1. 현재 문제점

### 1.1 미승인 단어로 용어 등록 가능
- 단어가 `APRV_YN = 'N'` (미승인) 상태인데도 용어 등록 시 해당 단어를 사용할 수 있음
- 표준 단어로 확정되지 않은 단어가 용어에 포함되면 표준 관리의 의미가 없어짐

### 1.2 모든 등록 경로에서 동일한 문제 발생

| 등록 경로 | 현재 동작 | 문제 |
|----------|---------|------|
| 용어 화면 등록 (createTerms) | 단어 존재만 체크 | 승인 여부 미체크 |
| 용어 엑셀 업로드 (uploadTermsList) | `selectWordByEngAbrvNm`으로 존재 체크 | 승인 여부 미체크 |
| 진단 결과 빠른 등록 | 기존 단어 참조 | 승인 여부 미체크 |
| 표준화 추천 일괄등록 (registerTermsBatch) | newWords를 먼저 INSERT 후 용어 등록 | 단어가 미승인 상태로 등록됨 |
| 표준화 추천 단어등록 (registerWord) | 직접 TB_WORD에 INSERT | 관리자면 자동 승인, 아니면 미승인 |

### 1.3 변경 이력이 승인 전에도 저장됨
- 등록/수정/삭제 API 호출 시 바로 이력 저장
- 승인 반려 시 무의미한 이력이 쌓임
- 정식 이력은 승인 완료 시점에만 저장되어야 함

---

## 2. 설계 원칙

### 2.1 승인된 단어만 용어에 사용 가능
- 용어 등록(모든 경로)에서 구성 단어의 `APRV_YN = 'Y'` 체크
- 미승인 단어가 포함되면 등록 거부 + 어떤 단어가 미승인인지 안내

### 2.2 변경 이력은 승인 완료 시점에만 저장
- 등록/수정/삭제 API에서 이력 저장 제거
- 승인 처리 API에서 승인 완료 시 이력 저장

### 2.3 관리자 즉시 승인 유지
- 관리자(`isAdmin`)가 등록하면 `APRV_YN = 'Y'`로 즉시 승인 → 이력도 즉시 저장
- 일반 사용자가 등록하면 `APRV_YN = 'N'` → 승인 후 이력 저장

## 3. 수정 대상

### 3.1 용어 등록 시 단어 승인 체크 추가

#### 3.1.1 createTerms (용어 화면 등록)

```
현재: 단어 영문약어 → selectWordByEngAbrvNm → 존재하면 OK
변경: 단어 영문약어 → selectWordByEngAbrvNm → 존재 + APRV_YN = 'Y' 확인
      → 미승인 단어 포함 시: "다음 단어가 아직 승인되지 않았습니다: XXX" 에러 반환
```

#### 3.1.2 uploadTermsList (용어 엑셀 업로드)

```
현재: wordVo = selectWordByEngAbrvNm(영문약어) → null이면 실패
변경: wordVo = selectWordByEngAbrvNm(영문약어) → null이면 실패
      → wordVo.getAprvYn() != 'Y' 이면 실패 ("단어 미승인: XXX")
```

#### 3.1.3 registerTermsBatch (표준화 추천 일괄등록)

```
현재: newWords를 TB_WORD에 INSERT → 바로 용어 등록
변경:
  - 관리자: newWords INSERT 시 APRV_YN = 'Y' (기존 동작 유지)
  - 일반 사용자: newWords INSERT 시 APRV_YN = 'N'
    → 용어 등록 시 미승인 단어 체크 → 실패
    → "먼저 단어 승인을 받은 후 용어를 등록해주세요" 안내
```

#### 3.1.4 registerWord (표준화 추천 단어 단건 등록)

```
현재: 관리자면 APRV_YN = 'Y', 아니면 미설정
변경: 관리자면 APRV_YN = 'Y', 일반 사용자면 APRV_YN = 'N' (명시적)
```

### 3.2 변경 이력 저장 시점 변경

#### 현재 흐름
```
사용자 → createWord → TB_WORD INSERT + 이력 저장
                                        ↑ 문제: 아직 승인 안 됨
```

#### 변경 흐름
```
사용자 → createWord → TB_WORD INSERT (APRV_YN = 'N')
관리자 → 승인 처리 → APRV_YN = 'Y' 업데이트 + 이력 저장
                                                ↑ 승인 시점에 이력
```

#### 관리자 직접 등록 시
```
관리자 → createWord → TB_WORD INSERT (APRV_YN = 'Y') + 이력 저장
                                                       ↑ 즉시 승인이므로 바로 이력
```

### 3.3 승인 처리 API에 이력 저장 추가

승인 처리 관련 API (approve.xml / DataStandardController 또는 별도 컨트롤러):
- 승인 완료 시: 해당 항목의 이력을 TB_CHANGE_HISTORY에 INSERT
- 반려 시: 이력 저장 안 함

## 5. 승인 프로세스 전체 흐름 (변경 후)

### 5.1 단어 등록 → 승인 → 용어 등록

```
일반 사용자:
  1. 단어 등록 요청 → TB_WORD INSERT (APRV_YN = 'N')
  2. 관리자 승인 대기
  3. 관리자 승인 → APRV_YN = 'Y' + 이력 저장 (INSERT)
  4. 승인된 단어로 용어 등록 가능

관리자:
  1. 단어 등록 → TB_WORD INSERT (APRV_YN = 'Y') + 이력 저장
  2. 바로 용어 등록 가능
```

### 5.2 표준화 추천 흐름

```
일반 사용자:
  1. 표준화 추천에서 단어 등록 → APRV_YN = 'N'
  2. "확인" 클릭 → 미승인 단어 경고 → 용어 등록 불가
  3. 관리자 승인 후 다시 등록 시도

관리자:
  1. 표준화 추천에서 단어 등록 → APRV_YN = 'Y'
  2. "확인" 클릭 → 승인된 단어이므로 용어 등록 가능
  3. 일괄 등록 완료 + 이력 저장
```

## 6. 구현 우선순위

| 순서 | 항목 | 필수 여부 |
|------|------|----------|
| 1 | 용어 등록 시 단어 승인 체크 (createTerms, uploadTermsList) | 필수 |
| 2 | registerTermsBatch에 단어 승인 체크 | 필수 |
| 3 | registerWord에 일반 사용자 APRV_YN = 'N' 명시 | 필수 |
| 4 | 개별 등록 API에서 이력 저장 제거 (관리자 제외) | 필수 |
| 5 | 승인 처리 API에 이력 저장 추가 | 필수 |
| 6 | 프론트 에러 메시지 ("단어 미승인") | 필수 |
| 7 | 반려 시 이력 미저장 확인 | 권장 |
