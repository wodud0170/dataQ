# 표준화 자동 추천 등록 시스템 설계

> 작성일: 2026-03-27
> 목적: 개발자가 엑셀로 한글 컬럼명을 입력하면, 표준 용어/단어를 자동 분석하여 추천 및 일괄 등록하는 시스템

---

## 1. 문제 정의

### 현재 상황
```
개발자: 테이블에 "기준일자", "사용자코드명", "월별정산금액" 컬럼 추가하려는데
        이게 표준에 맞는지 모르겠어요
```

### 현재 워크플로우 (비효율)
```
1. 데이터 표준 담당자에게 문의
2. 담당자가 용어사전 수동 검색
3. 없으면 → 단어 등록 → 도메인 매칭 → 용어 등록 (3~5분/건)
4. 있으면 → 컬럼명·타입·길이 안내
5. 개발자가 DDL 작성
```

**100개 컬럼 추가 시 → 수일 소요**

### 목표 워크플로우
```
1. 개발자가 한글 컬럼명 엑셀 업로드
2. 시스템이 자동 분석 (5초)
3. 담당자가 추천 결과 리뷰 → 원클릭 승인
4. 표준 용어 일괄 등록 + DDL 자동 생성
```

**100개 컬럼 → 5분 이내 완료**

### Step 2: 자동 분석 (핵심 엔진)

#### 3-2-1. 분석 파이프라인

```
한글명 "사용자코드명"
    │
    ▼
[1단계] 기존 용어 완전매칭 확인
    │   → TB_TERMS.TERMS_NM = '사용자코드명' ?
    │   → 있으면 ✅ "기등록 용어" 상태로 즉시 반환
    │
    ▼ (없으면)
[2단계] 형태소 분석 (OKT)
    │   → "사용자" + "코드" + "명"
    │
    ▼
[3단계] 단어 매칭
    │   "사용자" → TB_WORD 조회 → [USER(사용자), USAGER(사용자)] 후보
    │   "코드"   → TB_WORD 조회 → [CD(코드)] 후보
    │   "명"     → TB_WORD 조회 → [NM(명)] 후보
    │
    ▼
[4단계] 최적 단어 선택 (자동 추천)
    │   각 단어 후보 중 1개를 자동 선택
    │   선택 기준: 사용 빈도 > 영문약어 길이(짧을수록) > 등록일(최신)
    │
    ▼
[5단계] 영문약어 조합
    │   USER + CD + NM → "USER_CD_NM"
    │
    ▼
[6단계] 도메인 자동 추천
    │   마지막 단어 "명(NM)" → domainClsfNm 조회
    │   → 해당 분류의 도메인 목록 반환
    │   → 1건이면 자동 선택, 복수면 사용 빈도순 추천
    │
    ▼
[7단계] 결과 상태 판정
    ├── ✅ 완전매칭: 기등록 용어 (변경 불필요)
    ├── 🟢 자동완성: 모든 단어 매칭 + 도메인 추천 성공
    ├── 🟡 부분매칭: 일부 단어 미등록 (단어 등록 필요)
    └── 🔴 미매칭:   형태소 분석 실패 또는 대부분 미등록
```

#### 3-2-2. 단어 자동 선택 알고리즘

```java
// 우선순위 스코어링
int score(StdWordVo word) {
    int s = 0;
    // 1. 용어에서 사용된 빈도 (TB_TERMS_WORDS에 등장 횟수)
    s += getUsageCount(word.getId()) * 100;
    // 2. 영문약어가 짧을수록 높은 점수 (관례상 짧은 게 표준)
    s += (10 - word.getWordEngAbrvNm().length()) * 10;
    // 3. 공통표준여부 Y이면 가산
    if ("Y".equals(word.getCommStndYn())) s += 50;
    return s;
}
```

| 후보          | 빈도  | 약어길이 | 공통표준 | 점수              | 선택  |
| ----------- | --- | ---- | ---- | --------------- | --- |
| USER(사용자)   | 45  | 4    | Y    | 4500+60+50=4610 | ✅   |
| USAGER(사용자) | 2   | 6    | N    | 200+40+0=240    |     |
|             |     |      |      |                 |     |

#### 3-2-3. 미등록 단어 처리

미등록 단어 발견 시 **자동 추천값**을 생성:

```
한글명: "정산"  (미등록)
    │
    ▼
[추천 생성]
    영문약어 추천: 기존 단어 패턴 분석
        - 2글자 한글 → 보통 2~4자 영문
        - "정산" → "STLM" (Settlement 약어 추천)
        - 후보: STLM, STTL, ADJST
    영문명 추천: "Settlement"
    도메인분류 추천: 마지막 글자 기반
        - "금" → 금액, "일"/"자" → 날짜, "명" → 명칭
        - 없으면 → "기타"

결과:
    ┌ 단어 한글명: 정산          (확정)
    │ 영문약어:    STLM          (추천, 수정 가능)
    │ 영문명:      Settlement    (추천, 수정 가능)
    └ 도메인분류:  기타          (추천, 수정 가능)
```

### Step 4: 일괄 등록 + DDL 생성

#### 3-4-1. 등록 프로세스

```

    │
    ▼

    │   ┌─────────────────────────┐
    │   │ 등록 완료               │
    │   │ 단어 신규등록: 1건      │
```

### 5-2. TermRecommendService 핵심 로직

```java
public class TermRecommendService {

    /**
     * 핵심 메서드: 한글명 리스트 → 분석 결과
     */
    public List<TermAnalysisResult> analyze(List<String> termNames) {
        // 1. 기등록 용어 일괄 조회 (1 query)
        Map<String, StdTermsVo> existingTerms = loadExistingTerms();

        // 2. 기등록 단어 전체 캐시 (1 query)
        Map<String, List<StdWordVo>> wordCache = loadAllWordsByNm();

        // 3. 단어 사용빈도 조회 (1 query)
        Map<String, Integer> wordUsageCount = loadWordUsageCounts();

        // 4. 도메인 전체 캐시 (1 query)
        Map<String, List<StdDomainVo>> domainByClsf = loadDomainsByClsf();

        List<TermAnalysisResult> results = new ArrayList<>();

        for (String name : termNames) {
            // 4-1. 기등록 체크
            if (existingTerms.containsKey(name)) {
                results.add(buildRegisteredResult(name, existingTerms.get(name)));
                continue;
            }

            // 4-2. 형태소 분석
            Map<String, String> tokens = StringWordAnalyzer
                .getWordsFromStringByOkt(name, null, false);

            // 4-3. 각 토큰별 단어 매칭 + 스코어링
            List<WordAnalysis> wordAnalyses = new ArrayList<>();
            for (String token : tokens.keySet()) {
                List<StdWordVo> candidates = wordCache.get(token);
                if (candidates != null && !candidates.isEmpty()) {
                    // 스코어링 후 정렬
                    scoreCandidates(candidates, wordUsageCount);
                    wordAnalyses.add(new WordAnalysis(token, "MATCHED", candidates));
                } else {
                    // 미등록 → 추천값 생성
                    StdWordVo suggested = suggestNewWord(token);
                    wordAnalyses.add(new WordAnalysis(token, "NEW", suggested));
                }
            }

            // 4-4. 도메인 추천 (마지막 단어 기준)
            String lastWordClsf = getLastWordDomainClsf(wordAnalyses);
            List<StdDomainVo> domainCandidates = domainByClsf.get(lastWordClsf);

            // 4-5. 상태 판정
            String status = determineStatus(wordAnalyses);

            results.add(buildResult(name, status, wordAnalyses, domainCandidates));
        }

        return results;
    }
}
```

### 5-3. 필요한 신규 쿼리 (terms.xml / word.xml)

```xml
<!-- 단어 사용빈도 조회 -->
<select id="selectWordUsageCounts" resultType="map">
    SELECT W.WORD_NM, COUNT(*) as CNT
    FROM TB_WORD W
    INNER JOIN TB_TERMS_WORDS TW ON W.WORD_ID = TW.WORD_ID
    GROUP BY W.WORD_NM
</select>

<!-- 전체 단어를 한글명 기준으로 조회 (캐시용) -->
<select id="selectAllWordsGroupByNm" resultMap="stdWordMap">
    SELECT * FROM TB_WORD ORDER BY WORD_NM
</select>

<!-- 전체 용어를 한글명 기준으로 조회 (캐시용) -->
<select id="selectAllTermsByNm" resultMap="stdTermsMap">
    SELECT T.*, D.DATA_TYPE, D.DATA_LEN
    FROM TB_TERMS T
    LEFT JOIN TB_DOMAIN D ON T.DOMAIN_NM = D.DOMAIN_NM
</select>
```

## 9. 확장 가능성

### Phase 2: 학습형 추천
- 사용자가 추천을 수정한 이력을 저장
- 수정 패턴을 학습하여 추천 정확도 향상
- 예: "정산" → 처음엔 STLM 추천, 사용자가 ADJST로 수정 → 다음부터 ADJST 우선 추천

### Phase 3: 데이터소스 연동
- 기존 테이블 스키마를 자동 스캔
- 비표준 컬럼 자동 감지 → 표준화 추천
- 진단 결과 화면과 통합

### Phase 4: API 연동
- 외부 시스템에서 REST API로 표준화 검증 요청
- CI/CD 파이프라인에 DDL 표준화 검증 단계 추가
