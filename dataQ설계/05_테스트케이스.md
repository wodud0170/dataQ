# 테스트 케이스

## 1. 용어 빠른 등록 (진단 결과 > 컬럼 상세)

### TC-1-1. 일반 도메인으로 용어 등록
| 항목 | 내용 |
|------|------|
| 사전조건 | 진단 완료, 컬럼 상세 탭에 TERM_NOT_EXIST 이슈 존재 |
| 절차 | 1. 컬럼 상세에서 "용어 미존재" 행의 [용어 등록] 클릭<br/>2. 단어 분석 확인 (등록됨/미등록 표시)<br/>3. 도메인 유형: **일반 도메인** 선택 (기본값)<br/>4. 도메인 autocomplete에서 선택<br/>5. 용어 한글명, 설명 입력<br/>6. [용어 등록] 클릭 |
| 기대결과 | - 용어가 TB_TERMS에 등록됨 (CODE_GRP = null)<br/>- "재진단 시 반영됩니다" 알림 표시<br/>- 재진단 후 해당 컬럼의 TERM_NOT_EXIST 이슈 해소 |

### TC-1-2. 코드로 용어 등록
| 항목 | 내용 |
|------|------|
| 사전조건 | 코드가 1건 이상 등록되어 있어야 함 |
| 절차 | 1. [용어 등록] 클릭<br/>2. 도메인 유형: **코드** 선택<br/>3. 코드 autocomplete에서 코드 선택<br/>4. 코드의 도메인 정보(타입/길이) 표시 확인<br/>5. [용어 등록] 클릭 |
| 기대결과 | - TB_TERMS에 등록됨<br/>- domainNm = 선택한 코드의 도메인명<br/>- codeGrp = 선택한 코드의 코드그룹<br/>- 재진단 후 타입/길이도 코드 도메인 기준으로 검증됨 |

### TC-1-3. 미등록 단어 인라인 등록 후 용어 등록
| 항목 | 내용 |
|------|------|
| 사전조건 | 컬럼 영문명에 미등록 단어가 포함됨 (예: 새로운 약어) |
| 절차 | 1. [용어 등록] 클릭<br/>2. 단어 분석에서 "미등록" 단어 확인<br/>3. 한글명/영문명 입력 후 [단어 등록] 클릭<br/>4. 상태가 "등록됨"으로 변경 확인<br/>5. 도메인 선택 → 용어 등록 |
| 기대결과 | - TB_WORD에 단어 등록됨<br/>- 단어 등록 후 용어 등록 버튼 활성화<br/>- 용어 정상 등록 |

### TC-1-4. 필수값 미입력 시 등록 버튼 비활성화
| 항목 | 내용 |
|------|------|
| 절차 | 1. 용어 한글명 비우기 → 등록 버튼 비활성화 확인<br/>2. 도메인 미선택 → 비활성화 확인<br/>3. 코드 모드에서 코드 미선택 → 비활성화 확인<br/>4. 미등록 단어 있을 때 → 비활성화 확인 |
| 기대결과 | 모든 필수값 입력 전까지 [용어 등록] 버튼 disabled |

## 2. 용어 등록 화면 (DSTerm.vue)

### TC-2-1. 일반 도메인으로 등록
| 항목 | 내용 |
|------|------|
| 절차 | 1. Step 1: 용어명 입력<br/>2. Step 2: 단어 선택<br/>3. Step 3: 도메인 유형 = 일반 도메인 (기본값)<br/>4. 도메인 autocomplete에서 선택<br/>5. 설명 입력 → 저장 |
| 기대결과 | - 기존과 동일하게 동작<br/>- CODE_GRP = null |

### TC-2-2. 코드로 등록
| 항목 | 내용 |
|------|------|
| 절차 | 1. Step 3에서 도메인 유형 = **코드** 선택<br/>2. 코드 목록 로드 확인<br/>3. 코드 autocomplete에서 선택<br/>4. 코드의 도메인 정보 표시 확인<br/>5. 코드그룹 필드가 숨겨짐 확인<br/>6. 저장 |
| 기대결과 | - domainNm = 코드의 도메인명<br/>- codeGrp = 코드의 코드그룹<br/>- 코드그룹 필드 미노출 |

### TC-2-3. 도메인 유형 전환
| 항목 | 내용 |
|------|------|
| 절차 | 1. 일반 도메인 선택 후 도메인 선택<br/>2. 코드로 전환<br/>3. 코드 선택<br/>4. 다시 일반 도메인으로 전환 |
| 기대결과 | - 전환 시 UI가 정상 변경됨<br/>- 코드그룹 필드 show/hide 정상 동작 |

## 3. 진단 로직 변경

### TC-3-1. 타입 동의어 처리
| 표준 타입 | 실제 타입 | 기대결과 |
|----------|----------|---------|
| VARCHAR | VARCHAR2 | 일치 (이슈 없음) |
| VARCHAR2 | VARCHAR | 일치 |
| CHAR | VARCHAR | 일치 |
| CHAR | VARCHAR2 | 일치 |
| DATE | DATETIME | 일치 |
| DATETIME | DATE | 일치 |
| NUMBER | NUMERIC | 일치 |
| NUMBER | DECIMAL | 일치 |
| NUMERIC | DECIMAL | 일치 |
| VARCHAR | NUMBER | **불일치** (DATA_TYPE_MISMATCH) |
| DATE | VARCHAR | **불일치** |

### TC-3-2. 소수점 비교 제거
| 표준 | 실제 | 기대결과 |
|------|------|---------|
| NUMERIC(15,2) | NUMERIC(15) | 일치 (길이 15 동일, 소수점 무시) |
| NUMERIC(15,2) | NUMERIC(15,0) | 일치 |
| NUMERIC(15,2) | NUMERIC(10) | **불일치** (길이 다름) |
| NUMBER(10,2) | NUMBER(10,5) | 일치 (길이 10 동일, 소수점 무시) |

### TC-3-3. 길이만 비교
| 표준 길이 | 실제 길이 | 기대결과 |
|----------|----------|---------|
| 100 | 100 | 일치 |
| 100 | 200 | **불일치** (DATA_LEN_MISMATCH) |
| 0 (미설정) | 100 | 일치 (표준 길이 0이면 비교 안함) |

## 4. 코맨트/MODIFY 스크립트 생성

### TC-4-1. COMMENT ON 스크립트 (한글명 불일치)
| 항목 | 내용 |
|------|------|
| 사전조건 | TERM_KR_NM_MISMATCH 이슈가 있는 행 |
| 절차 | 1. [코맨트 변경] 클릭<br/>2. DBMS에 맞는 COMMENT 스크립트 표시 확인<br/>3. [복사] 클릭 |
| 기대결과 | - Oracle: `COMMENT ON COLUMN 테이블.컬럼 IS '표준명';`<br/>- 복사 성공 시 "복사되었습니다" 표시 |

### TC-4-2. ALTER MODIFY 스크립트 (타입/길이 불일치)
| 항목 | 내용 |
|------|------|
| 사전조건 | DATA_TYPE_MISMATCH 또는 DATA_LEN_MISMATCH 이슈가 있는 행 |
| 절차 | 1. [컬럼 변경] 클릭<br/>2. ALTER TABLE MODIFY 스크립트 표시 확인<br/>3. [복사] 클릭 |
| 기대결과 | - Oracle: `ALTER TABLE 테이블 MODIFY 컬럼 타입(길이);`<br/>- 백업 권장 경고 문구 포함 |

## 5. 이슈 유형별 드릴다운 (진단 결과 탭)

### TC-5-1. 이슈 유형 칩 클릭 → 컬럼 상세 이동
| 항목 | 내용 |
|------|------|
| 절차 | 1. 진단 결과 탭의 이슈 유형별 분석에서 칩 클릭 (예: 용어 미존재)<br/>2. 컬럼 상세 탭으로 이동 확인<br/>3. 진단유형 필터에 해당 유형만 체크 확인 |
| 기대결과 | - activeTab이 컬럼 상세로 변경<br/>- selectedDiagTypes에 클릭한 유형만 설정<br/>- 필터링된 결과만 표시 |
