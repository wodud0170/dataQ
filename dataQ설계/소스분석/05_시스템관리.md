# 소스분석 - 시스템 관리

> 최종 수정: 2026-03-23

## 개요

수집 대상 DB의 접속 정보(데이터소스)와 시스템 정보를 관리하는 기능.
데이터모델 수집의 전제 조건으로, 데이터소스 없이는 수집 불가.

## 기능 구성

```
시스템 관리
├── 시스템 정보 (MMSystem.vue)       TB_SYS_INFO
├── 데이터소스 (MMDatasource.vue)    TB_DATA_SOURCE
└── 사용자 관리 (MMUser.vue)         (별도 인증 테이블)
```

---

## 데이터소스 (TB_DATA_SOURCE)

### 역할
- 수집 대상 외부 DB의 접속 정보 저장
- q-executor에서 DataSourceUtils로 JDBC 연결 시 사용

### 주요 기능
| 기능 | 설명 |
|------|------|
| 등록/수정/삭제 | CRUD |
| 연결 테스트 | `/api/sysinfo/validateDataSource` → 실제 JDBC 연결 시도 |
| 비밀번호 암호화 | Jasypt로 암호화하여 TB_DATA_SOURCE.PWD에 저장 |

### 지원 DBMS
| DBMS 코드 | 설명 |
|-----------|------|
| oracle | Oracle DB |
| postgresql | PostgreSQL |
| mysql | MySQL |
| mariadb | MariaDB |
| mssql | Microsoft SQL Server |
| cubrid | Cubrid |

### 보안
- PWD 컬럼은 Jasypt 암호화 저장
- SecurityManager(common lib)로 암호화/복호화 처리
- 화면에서 비밀번호 마스킹 처리 필요 (현황 미확인)

---

## 시스템 정보 (TB_SYS_INFO)

### 역할
- 데이터모델과 연결되는 시스템(업무 시스템) 정보 관리
- `DM_SYS_CD` 로 데이터모델과 연결

### 주요 필드
| 필드 | 설명 |
|------|------|
| SYS_CD | 시스템 코드 (PK) |
| SYS_NM | 시스템명 |
| SYS_DESC | 설명 |

---

## 사용자 관리 (MMUser.vue)

### 역할
시스템 접속 계정 관리.

### 현재 상태
- 화면 존재하나 구현 완성도 낮음
- 인증은 `CustomAuthenticationProvider`에서 DB 조회로 처리
- 사용자 테이블 구조 별도 확인 필요 (공통 lib 내 포함 가능성)

---

## 현재 한계 / 개선 여지

| 항목 | 현황 | 개선 방향 |
|------|------|----------|
| 비밀번호 마스킹 | 화면에서 마스킹 처리 여부 불명확 | 수정 화면에서 비밀번호 재입력 방식으로 변경 |
| 연결 테스트 | 단순 연결 성공/실패만 반환 | 연결 지연시간, 버전 정보 등 추가 반환 |
| 데이터소스 복사 | 없음 | 유사 설정 복사 후 수정 기능 |
| 접속 이력 | 없음 | 수집 시 사용된 접속 이력 로깅 |
| 사용자 역할 관리 | MMRoles.vue 숨김 처리 | 역할 기반 접근 제어(RBAC) 구현 |
