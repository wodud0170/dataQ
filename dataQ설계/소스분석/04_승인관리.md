# 소스분석 - 승인 관리

> 최종 수정: 2026-03-23

## 개요

표준 사전(용어·단어·도메인) 등록/수정 시 승인 프로세스를 통해 품질을 관리하는 기능.
현재 구조는 기반은 갖춰져 있으나 완성도가 낮고, 실제 워크플로우 연결이 미흡하다.

## 관련 파일

| 파일                          | 역할               |
| --------------------------- | ---------------- |
| `MMApproval.vue`            | 승인 관리 화면         |
| `TB_APPROVE_STAT`           | 승인 상태 이력 테이블     |
| `approve.xml`               | 승인 관련 MyBatis 쿼리 |
| `NDQualityApproveStat.java` | 승인 상태 열거형        |
| `StdApproveStatVo.java`     | 승인 상태 VO         |

## 현재 구현 상태

### TB_APPROVE_STAT 구조
```
APPROVE_ID  - 승인 ID
OBJ_TYPE    - 객체 타입 (DOMAIN / TERMS / WORD 등)
OBJ_ID      - 객체 ID
APRV_YN     - 승인 여부 (Y/N)
APRV_USER_ID - 승인자 ID
APRV_DT     - 승인 일시
```

### 각 표준 객체의 승인 필드
- TB_DOMAIN, TB_TERMS, TB_WORD 모두 `APRV_YN` 컬럼 보유
- 미승인(N) 상태로 등록 후 승인(Y)으로 변경하는 흐름 의도

### 승인 화면 (MMApproval.vue)
- 미승인 항목 목록 조회
- 승인/반려 처리
- 대시보드(QDashboard.vue)에서 미승인 건수 표시 및 연동

---

## 현재 한계

| 항목 | 현황 |
|------|------|
| 승인 워크플로우 | 등록자 → 승인자 다단계 프로세스 미구현 |
| 승인 알림 | 승인 요청 시 알림(WebSocket/이메일) 없음 |
| 반려 사유 | 반려 시 사유 기록 필드 없음 |
| 승인 권한 | 역할(Role) 기반 승인 권한 분리 미구현 |
| 일괄 승인 | 개별 승인만 가능, 다건 일괄 승인 없음 |

---

## 개선 방향

1. **역할 기반 승인 권한**: MMRoles.vue와 연계하여 승인자 역할 지정
2. **승인 요청 알림**: 등록 시 WebSocket으로 승인자에게 실시간 알림
3. **반려 사유 필드**: TB_APPROVE_STAT에 REJECT_REASON 컬럼 추가
4. **이력 관리**: 승인/반려 이력 전체 조회 기능
5. **일괄 승인**: 체크박스 선택 후 일괄 승인/반려
