# 용어(Terms) 시스템 분석

## 1. 전체 구조 (ER 관계)

```
TB_WORD (단어)
  ├── wordNm (한글명)
  ├── wordEngAbrvNm (영문약어) ← 용어 영문명 생성에 사용
  └── domainClsfNm (도메인분류명) ← 도메인 필터링에 사용

TB_TERMS (용어)
  ├── termsNm (한글명)
  ├── termsEngAbrvNm (영문약어) = 단어들의 wordEngAbrvNm을 '_'로 연결
  └── domainNm (도메인명) → FK

TB_TERMS_WORDS (용어-단어 매핑)
  ├── termsId → FK (TB_TERMS)
  ├── wordId → FK (TB_WORD)
  └── wordOrd (순서)

TB_DOMAIN (도메인)
  ├── domainNm (도메인명) ← 용어에서 참조
  ├── dataType, dataLen, dataDecimalLen ← 진단 시 비교 기준
  └── domainClsfNm (분류명) ← 단어의 domainClsfNm과 연결
```

### 관계 요약
```
단어(N) ──매핑──> 용어(1) ──참조──> 도메인(1)
  │                                    │
  └── domainClsfNm으로 필터링 ──────────┘
```

---

## 2. 용어 영문약어명(termsEngAbrvNm) 생성 규칙

**핵심: 컬럼명과 용어의 매칭 키**

```
단어 선택: [기준(BASS), 일자(DT)]
    ↓
영문약어 연결: BASS + '_' + DT
    ↓
termsEngAbrvNm = "BASS_DT"
```

| 선택된 단어 | 각 약어 | 결과 |
|-----------|--------|------|
| 기준, 일자 | BASS, DT | BASS_DT |
| 건물, 명 | BLDG, NM | BLDG_NM |
| 등록, 일시 | REGIST, DT | REGIST_DT |

**진단 매칭**: 컬럼 영문명(attrNm) == termsEngAbrvNm → 용어 존재

## 3. 용어 등록 프로세스 (DSTerm.vue)

### 3-Step Stepper

```
┌──────────────────────────────────────────────────┐
│ Step 1: 용어명 입력                                │
│   "기준일자" 입력                                   │
│   → 형태소 분석 (OKT) → [기준, 일자] 추출            │
├──────────────────────────────────────────────────┤
│ Step 2: 단어 선택                                  │
│   "기준" → DB 조회 → [기준(BASS), 기준(STD), ...]   │
│   "일자" → DB 조회 → [일자(DT), 일자(DD), ...]      │
│   사용자가 각 그룹에서 1개씩 선택                      │
├──────────────────────────────────────────────────┤
│ Step 3: 용어 정보 입력                              │
│   영문약어: BASS_DT (자동, readonly)                │
│   도메인: [일자(DT)의 domainClsfNm으로 필터링]        │
│   설명: (필수 입력)                                 │
│   기타: 이음동의어, 코드그룹, 담당기관 등 (선택)         │
└──────────────────────────────────────────────────┘
```

### 핵심 의존관계 (문제점)

```
용어 등록하려면
  └── 단어가 먼저 있어야 함 (Step 2에서 선택)
        └── 없으면 단어 등록 화면으로 이동해야 함
              └── 단어 등록 시 영문약어, 도메인분류 등 입력 필요
                    └── 도메인이 먼저 있어야 도메인분류 참조 가능
```

**체인 문제**: 용어 등록 → 단어 없음 → 단어 등록 → 도메인 없음 → 도메인 등록 → 다시 단어 → 다시 용어

## 5. 진단 매칭 로직 (DiagService.java)

```
1. 전체 용어를 메모리 로드
   Map<termsEngAbrvNm, StdTermsVo> (도메인 LEFT JOIN)

2. 각 컬럼 순회:
   attrNm(컬럼 영문명)으로 용어 조회
     ├── 미존재 → TERM_NOT_EXIST
     └── 존재
           ├── 한글명 비교 → 불일치 시 TERM_KR_NM_MISMATCH
           └── 도메인 있으면
                 ├── 타입 비교 → 불일치 시 DATA_TYPE_MISMATCH
                 └── 길이 비교 → 불일치 시 DATA_LEN_MISMATCH
```

### 타입 동의어 처리
| 그룹 | 동일 취급 타입 |
|------|-------------|
| 문자 | CHAR, VARCHAR, VARCHAR2 |
| 숫자 | NUMBER, NUMERIC, DECIMAL |
| 날짜 | DATE, DATETIME |
