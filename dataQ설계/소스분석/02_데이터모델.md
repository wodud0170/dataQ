# 소스분석 - 데이터모델

> 최종 수정: 2026-03-23

## 개요

외부 DB에서 테이블/컬럼 정보를 수집하고, 표준 사전 기준으로 표준화율을 산출하여 현황을 조회하는 기능.

## 기능 구성

```
데이터모델
├── 수집 (DSDatamodelCollection.vue)    수집 실행 및 스키마 필터 설정
├── 수집이력 (DSDatamodelHistory.vue)   수집이력 목록 조회
├── 현황 (DSDatamodelStatus.vue)        최신 수집 기준 통계 대시보드
│   ├── 테이블 (DSDatamodelStatusTable.vue)    테이블 목록
│   └── 컬럼 (DSDatamodelStatusColumn.vue)     컬럼 목록
```

## 데이터 구조

```
TB_DATA_MODEL (데이터모델 정의)
  └── TB_DATA_MODEL_SCHEMA (수집 대상 스키마 필터)
  └── TB_DATA_MODEL_CLCT (수집이력)
        ├── TB_DATA_MODEL_OBJ (수집된 테이블 목록)
        ├── TB_DATA_MODEL_ATTR (수집된 컬럼 목록)
        └── TB_DATA_MODEL_STATS (표준화율 통계)
```

---

## 수집 기능

### 흐름

```
1. DSDatamodelCollection.vue: 데이터모델 선택 + 스키마 필터 설정
2. "수집 시작" 버튼 클릭
   → POST /api/dm/collectDataModel { dmId }
3. q-center DataModelController:
   → TB_DATA_MODEL_CLCT INSERT (수집이력 생성)
   → POST q-executor /api/dm/collectDataModel (WebClient, 비동기)
   → 즉시 HTTP 응답 반환 (수집은 백그라운드 진행)
4. q-executor DataModelService (별도 스레드):
   → TB_DATA_SOURCE에서 접속 정보 조회
   → DataSourceUtils로 외부 DB JDBC 연결
   → TB_DATA_MODEL_SCHEMA에서 스키마 필터 조회
   → DBMS별 SQL로 테이블/컬럼 조회
   → TB_DATA_MODEL_OBJ batch insert
   → TB_DATA_MODEL_ATTR batch insert
   → 표준화율 계산 → TB_DATA_MODEL_STATS insert
   → StompSessionService로 완료 메시지 전송 (WebSocket)
```

### DBMS별 수집 SQL 분기
| DBMS | 테이블 조회 | 컬럼 조회 |
|------|-----------|---------|
| Oracle | ALL_TABLES | ALL_TAB_COLUMNS |
| PostgreSQL | information_schema.tables | information_schema.columns |
| MySQL/MariaDB | information_schema.tables | information_schema.columns |
| MSSQL | sys.tables | sys.columns |
| Cubrid | db_class | db_attribute |

### 스키마 필터
- TB_DATA_MODEL_SCHEMA에 수집 대상 스키마명 저장
- 미설정 시 전체 스키마 수집
- UI에서 다중 스키마 선택 가능

---

## 수집이력 (DSDatamodelHistory.vue)

- TB_DATA_MODEL_CLCT 기반으로 수집 이력 목록 표시
- 수집 시작일시, 종료일시, 완성 여부 조회
- 특정 수집이력 선택 시 해당 시점의 테이블/컬럼 정보 조회 가능

---

## 현황 (DSDatamodelStatus.vue)

### 표시 정보
- 최신 수집 기준 통계 (TB_DATA_MODEL_STATS)
- 전체 표준화율, 용어 표준화율, 단어 표준화율, 도메인 표준화율
- 테이블 수, 컬럼 수

### 하위 화면 연계
```
DSDatamodelStatus (현황 선택)
  → 테이블명 클릭 → DSDatamodelStatusTable 탭 오픈
  → 컬럼 클릭 → DSDatamodelStatusColumn 탭 오픈
```

---

## 표준화율 계산 방식

```
용어 표준화율 = TERMS_STND_YN = 'Y'인 컬럼 수 / 전체 컬럼 수 × 100
단어 표준화율 = 단어 기준 일치 컬럼 수 / 전체 컬럼 수 × 100
도메인 표준화율 = 도메인 연결된 컬럼 수 / 전체 컬럼 수 × 100
전체 표준화율 = 세 비율의 평균
```

> **주의**: 표준화율은 수집 시점에 계산되어 TB_DATA_MODEL_STATS에 저장.
> 이후 표준 사전이 변경되어도 재수집 전까지 수치는 변하지 않음.

## 현재 한계 / 개선 여지

| 항목 | 현황 | 개선 방향 |
|------|------|----------|
| 수집 진행률 표시 | WebSocket으로 완료 메시지만 전송, 중간 진행률 없음 | 진단처럼 processCnt/totalCnt 폴링 방식으로 변경 |
| 수집 중지 기능 | 없음 | 진단의 stop 패턴(ConcurrentHashMap + AtomicBoolean) 적용 |
| 표준화율 실시간 반영 | 재수집 필요 | 표준 사전 변경 시 표준화율 재계산 트리거 제공 |
| 대용량 DB 수집 | 테이블/컬럼이 많을 경우 처리 시간 길어짐 | 페이징 batch 수집, 청크 단위 처리 |
| 수집 스케줄링 | 수동 실행만 가능 | cron 기반 정기 자동 수집 기능 추가 |
